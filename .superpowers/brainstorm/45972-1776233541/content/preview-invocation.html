Preview Pane: How Content Gets In

Multiple triggers, one consistent pane

1

Auto-detect Dev Server URLs

When terminal output contains localhost:3000 or similar, show a toast: "Dev server detected -- Open in Preview?". Click to open. Also detect common frameworks: Next.js, Vite, Express startup messages.

PS F:\project> npm run dev
VITE v5.0.0 ready in 340ms
Local: http://localhost:5173/
◉ Dev server detected Open Preview
2

Click File Paths in Terminal

File paths in terminal output (already linkified by xterm WebLinksAddon) become clickable. .html/.pdf/.png/.jpg/.svg open in Preview. Other files open in File Editor. Right-click gives "Open in Preview" / "Open in Editor" choice.

✓ Created src/renderer/components/Dashboard.tsx
✓ Generated docs/api-reference.html ← click opens in Preview
✓ Saved screenshots/dashboard.png ← click opens in Preview
3

Views Menu: Manual URL / File

Session toolbar "Views" menu has "Preview" option. Opens pane with URL bar where user can type any URL or browse for a local file. Also accessible via Ctrl+Shift+P.

◉ Preview
http://localhost:5173
↻ Refresh
📁 Browse
4

From Diff Viewer: Preview Changed Files

When viewing diffs, HTML/image files get a "Preview" button next to the filename. Click to see the rendered result alongside the diff. Great for verifying CSS/HTML changes visually.

index.html +12
styles.css +5
◉ Preview 📄 Edit
5

Command Bar Integration

Custom commands can target the Preview pane. E.g., a "Preview Storybook" command that opens localhost:6006 in preview after running the command in partner terminal. Extends existing command target system (claude/partner/preview).

Recommendation: All of the above

These aren't mutually exclusive -- they're all entry points into the same Preview pane. Auto-detect handles the common case (dev servers), click-paths handle file previews, Views menu is the manual fallback, diff integration is the power feature, and Command Bar extends it for custom workflows.
