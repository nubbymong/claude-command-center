### `7e9bfef` — fixes + Excalidraw + webview schema
- Dropped the small globe SVG that rendered next to the colored dot on global-scoped command buttons (`CommandBar.tsx`). Read as a second dot at small sizes.
- Restyled the Partner/Claude toggle to match the Snap tool button: monochrome `bg-surface0/60 + border-surface1/80`, intrinsic width, 12×12 icon.
- **SSH paste-leak fix** (`pty-manager.ts`):
  - Old code chained `setupCmd && claudeCmd` in a single ConPTY write. The long base64 setup blob could buffer and arrive after claude had started, dumping the entire one-liner into Claude's prompt textarea.
  - New state machine writes each step independently. **Two-step** for plain SSH: host setup → claude. **Three-step** for postCommand (e.g. asustor docker): host setup → postCommand → container setup re-run → claude. Re-running setup inside the container is required because host `~/.claude` is generally not bind-mounted.
  - Each step gates on the `setup ok` sentinel from the remote node script and a fresh shell prompt before the next write.
  - Hard latch: once `claudeSent` is true, watching for `╭`/`╰` box-drawing or `❯ ` cursor permanently disables auto-writes. Backstop in case any state flag mis-evaluates.
  - Setup timeout (10 s): if `setup ok` doesn't arrive, log + abort. User lands at bash and can debug rather than silent breakage.
- **Excalidraw scratchpad**: `npm install @excalidraw/excalidraw`, lazy-loaded modal with PNG copy-to-clipboard, `Draw` button next to Snap in every session. `backgroundImage` prop reserved for the webview-freeze flow.
- **Webview schema**: `CustomCommand.webView?: { enabled: boolean; url: string }`. CommandDialog gained the "Launch webview on completion" checkbox + URL input. When enabled, target picker locks to Partner.

### `4a4a8c0` — webview runtime + hooks injection disabled
- **WebviewButton** (`WebviewButton.tsx`): tool button next to Snap/Excalidraw, hidden when `status='idle'`. Pulse colours: neutral (pending), green (available, first 2xx-3xx), red (timeout). Click toggles the pane.
- **webviewStore** (Zustand): per-session `{ status, currentUrl, isOpen }`. `pollUrlForContent()` helper drives the activation poller via the main-process HEAD probe (CORS-bypass).
- **WebviewPane** (`WebviewPane.tsx`): renderer placeholder div whose bounds are streamed to main via `ResizeObserver` + 500 ms tick. Toolbar: back/forward/reload/home/freeze/close. Reload uses `reloadIgnoringCache()` to honour the spec's "always do a hard refresh".
- **WebContentsView lifecycle** (`webview-manager.ts`): per-session view attached to BrowserWindow contentView at the renderer-reported bounds. `persist:webview-<sid>` partition so cookies + cache survive reloads but don't leak across sessions. zod-validated IPC handlers in `webview-handlers.ts`. `closeAllWebviews()` wired into `before-quit`.
- **URL polling**: `net.request` HEAD with 3 s per-attempt timeout, falls back to GET on 4xx (some servers reject HEAD with 405). Renderer polls every 1 s for up to 30 s.
- **Freeze flow**: `WebviewPane`'s Freeze button calls `webContents.capturePage()`, passes the PNG dataURL into `ExcalidrawModal` as `backgroundImage` — user annotates over the snapshot, copies to clipboard.
- **App.tsx**: webview pane is a third toggle alongside Claude/Partner. When `webviewStore.bySessionId[sid].isOpen` is true the webview pane displays in place of the active terminal pane.
- **CommandBar `sendCommand`**: forces target='partner' for webview-enabled commands, writes the prompt, kicks off `pollUrlForContent`, flips button state on resolution.
- **Hooks injection disabled** (`pty-manager.ts`): Live Activity feed UI was cut in `c957e5d` but we kept injecting `hooks` blocks into per-session settings, pointing at `http://localhost:<gw-port>`. With no consumer feature attached, every Pre/PostToolUse fired into the void; on SSH the `-R` reverse tunnel often can't be established (sshd policy), so every call logged ECONNREFUSED. Both branches now skip the injection. Gateway code stays intact for the next consumer to flip the gate back on.
