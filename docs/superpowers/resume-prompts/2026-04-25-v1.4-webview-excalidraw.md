## Research findings

### Embedding external URLs in Electron

Source: [Electron docs — Web Embeds](https://www.electronjs.org/docs/latest/tutorial/web-embeds), [BrowserView docs](https://www.electronjs.org/docs/latest/api/browser-view), [Figma's BrowserView post](https://www.figma.com/blog/introducing-browserview-for-electron/)

| Option | Status | Notes |
|---|---|---|
| `<webview>` tag | **Not recommended.** Chromium's webview is undergoing dramatic architectural changes; rendering / nav / event routing all unstable. Slower than `<iframe>`. | Avoid. |
| `BrowserView` | **Deprecated** in favour of `WebContentsView`. Was the recommended sandbox-safe option (Figma, Slack used it). | Avoid for new code. |
| `WebContentsView` | **Modern recommended path.** Lives in OS window hierarchy (like Chrome tabs), as fast as Chrome. Positioned relative to owning `BrowserWindow`. | **Use this.** |
| `<iframe>` | Can work for trusted sources; same-origin / CSP issues for arbitrary URLs. Web-view will mostly point at `localhost:*` dev servers, so iframe could work too. | Fallback / simpler option. |

**Decision:** start with `WebContentsView` for proper Chromium isolation and Chrome-tab-like performance. Falls under main-process API (must be created in main, positioned via IPC from renderer).

### Embedding Excalidraw

Source: [@excalidraw/excalidraw on npm](https://www.npmjs.com/package/@excalidraw/excalidraw), [Excalidraw integration docs](https://docs.excalidraw.com/docs/@excalidraw/excalidraw/integration), [excalidraw-electron-app reference](https://github.com/kauppilainen/excalidraw-electron-app)

- Package: `@excalidraw/excalidraw` — distributes as ESM, requires `react`/`react-dom` peer deps (we already have React 18). Compatible with React 17/18/19.
- Renders as a single React component `<Excalidraw />`, accepts `initialData` / `onChange` for state persistence.
- For overlay mode: pass `theme="dark"`, transparent background CSS, position absolute over the WebContentsView region.
- For standalone mode: render in a fullscreen modal sibling to the terminal pane.
- Persistence: `excalidrawAPI.getSceneElements()` + `getAppState()` → JSON → save to per-config file. Restore via `initialData={{elements, appState}}`.

## Architectural notes

- `WebContentsView` is a main-process object. The renderer can't construct one directly. Pattern: renderer renders an empty placeholder div with a known `data-webview-id`, sends bounds to main via IPC, main creates / repositions `WebContentsView` over that bounds. ResizeObserver on the placeholder → IPC bound updates.
- "Send to web view on success" hook needs to know command exit codes. We already have PTY exit-code reporting (`pty:exit:${sessionId}`); the launcher path that runs custom commands probably writes to that PTY too. Verify by reading `useCommandStore` action handler. The web-view refresh should be a side-effect inside that handler when `webView.enabled`.
- Pulsing: reuse the `attention-pulse` keyframe already defined in `Sidebar.tsx` (`@keyframes attention-pulse`) — extract to a shared CSS file so the web-view button can import it.
- Excalidraw overlay needs to NOT capture pointer events when toggled off, so the underlying webview is interactive.
- "Copy whole page to clipboard" probably wants both `innerText` (paste-into-prompt friendly) and `outerHTML` (full-fidelity). Start with innerText.

## Open questions to ask the user before implementing

1. **Freeze semantics**: pause animations only, or fully halt JS execution? `webContents.setAudioMuted` + `executeJavaScript` to pause `<video>` is the obvious step; deeper freeze (suspend renderer process) may not be feasible per webview.
2. **Refresh strategy**: hard reload (`reload({ ignoringCache: true })`) or full `loadURL` with cache-bust query param? Hard reload is simpler.
3. **Standalone Excalidraw vs overlay — same component?**: yes, but the user may want different default tool palettes for "annotation over website" vs "blank scratchpad". Ask.
4. **Where does the standalone Excalidraw render?**: inside the current session's terminal area (replacing the terminal temporarily) or as a new pane sibling? Storyboard opens as a modal — same here?
5. **Per-config storage path**: should the Excalidraw scene live in `<resources>/excalidraw/<configId>.json` or be embedded into the existing config file? Separate file is cleaner.
6. **Web-view security**: load arbitrary URLs the user enters → CSP / mixed-content / cookies? For local dev URLs (`localhost:*`) this is fine; for arbitrary public URLs we should consider sandbox restrictions.
