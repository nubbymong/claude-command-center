## Big picture

This session shipped the **manual SSH connection flow** — replaces the fragile auto-detection state machine with explicit user-gated stages:

```
SSH connects → bash prompt visible
   ↓
  [SshFlowOverlay shows in CENTER of terminal pane]
   ↓
  Stage 1: "Run post-connect command?" (if configured) OR "Launch Claude?"
   ↓ user clicks
  Stage 2 (postCommand path only): inner shell appears → "Inner shell ready — launch Claude?"
   ↓ user clicks
  Stage 3: writeContainerSetupCmd OR writeClaudeCmd
   ↓
  Claude UI detected → overlay disappears
```

**Three layers of resilience for state transitions:**
1. Push notification (`win.webContents.send` of `ssh:flowState:<sid>`).
2. Renderer polls `ssh.getState(sid)` every 500 ms while still `connecting` (catch-up if push missed).
3. **Idle-data fallback in main** — 1.5 s of no PTY data → advance state via the same routing the prompt regex would have used. Independent of PS1 styling. **This is the reliable path** in practice — the regex misses non-standard prompts.
